# 発注書向け Backlog カスタム属性案

## 方針

Backlog 課題は「起案管理」に寄せ、PDF 出力に必要な値だけを過不足なく持たせる。  
相手方マスタで持てる項目は partner 側へ寄せ、案件ごとに変わる値だけを課題のカスタム属性に置く。

## 推奨構成

### 1. 基本情報

| 項目 | 型 | 必須 | 用途 |
| --- | --- | --- | --- |
| 契約種別 | リスト | 必須 | `発注書` など |
| 発注番号 | 文字列 | 必須 | `PO-ARC-DOM-202603005` |
| 発行日 | 日付 | 必須 | 発注書の日付 |
| 件名 | 文字列 | 必須 | `『モンスタースラッシャー』イラスト制作業務` |
| 基本契約有無 | チェックボックス | 必須 | スポット約款添付の分岐 |
| 基本契約参照番号 | 文字列 | 条件付き | 基本契約ありの場合のみ |

### 2. 社内担当情報

| 項目 | 型 | 必須 | 用途 |
| --- | --- | --- | --- |
| 担当部署 | 文字列 | 必須 | PDF 表示 |
| 担当者名 | 文字列 | 必須 | PDF 表示 |
| 担当者電話番号 | 文字列 | 任意 | PDF 表示 |
| 担当者メール | 文字列 | 必須 | PDF 表示 |

### 3. 発注概要

| 項目 | 型 | 必須 | 用途 |
| --- | --- | --- | --- |
| 発注合計金額（税抜） | 数値 | 任意 | 参考入力。実際は明細合計で算出推奨 |
| 納期 | 日付 or 文字列 | 必須 | 単一明細時はそのまま表示 |
| 支払条件 | 文字列 | 必須 | `2026年6月20日支払` など |
| 契約期間 | 文字列 | 任意 | 備考へ流し込み |
| 作業開始日 | 日付 | 任意 | 備考へ流し込み |
| 備考自由記載 | 長文 | 任意 | 個別事情だけを記載 |
| 特約事項 | 長文 | 任意 | そのまま PDF へ表示 |

### 4. 承諾・署名制御

| 項目 | 型 | 必須 | 用途 |
| --- | --- | --- | --- |
| 発注書署名欄表示 | チェックボックス | 任意 | `show_order_sign_section` |
| 受領確認セクション表示 | チェックボックス | 任意 | `accept_method` 併用前提 |
| 受領署名欄表示 | チェックボックス | 任意 | `show_sign_section` |
| 承諾方法 | リスト or 文字列 | 任意 | `メール返信` など |
| 承諾返信期限 | 日付 | 任意 | 受領確認欄に表示 |
| 着手をもって承諾 | チェックボックス | 任意 | 条項分岐 |

### 5. 振込情報

相手方マスタで保持するのを推奨。課題側に持つ場合は以下。

| 項目 | 型 | 必須 | 用途 |
| --- | --- | --- | --- |
| 銀行名 | 文字列 | 任意 | 支払先情報 |
| 支店名 | 文字列 | 任意 | 支払先情報 |
| 口座種別 | リスト | 任意 | `普通` など |
| 口座番号 | 文字列 | 任意 | 支払先情報 |
| 口座名義（カナ） | 文字列 | 任意 | 支払先情報 |
| 振込手数料負担者 | リスト | 任意 | `発注者` / `受注者` |

### 6. 明細テーブル

Backlog 標準機能だけでは明細の複数行管理が弱いので、実務上は JSON / Markdown / 連携側整形のどれかに寄せる。

各明細の推奨項目:

| 項目 | 型 | 必須 | 用途 |
| --- | --- | --- | --- |
| 業務名 | 文字列 | 必須 | 明細のタイトル |
| 仕様・補足 | 長文 | 任意 | 複数行でそのまま表示 |
| 数量 | 数値 | 必須 | `23` など |
| 単価 | 数値 | 任意 | 金額算出に使用 |
| 金額（税抜） | 数値 | 必須 | 明細金額 |
| 支払方法 | リスト | 必須 | 固定、出来高など |
| 明細別納期 | 日付 or 文字列 | 任意 | 複数明細時の個別表示 |
| 明細別支払条件 | 文字列 | 任意 | 複数明細時の個別表示 |
| 知的財産権帰属 | リスト | 任意 | `発注者帰属` など |

## 今回の発注書に当てはめた最小セット

- 発注番号
- 発行日
- 件名
- 基本契約有無
- 担当部署
- 担当者名
- 担当者電話番号
- 担当者メール
- 納期
- 支払条件
- 特約事項
- 備考
- 振込手数料負担者
- 発注書署名欄表示
- 明細 JSON もしくは明細複数行入力

## 実装メモ

- 単一明細なら `納期` と `支払条件` はヘッダ部にそのまま出す。
- 複数明細ならヘッダ部は `業務明細参照` にし、明細側へ個別値を出す。
- 備考は `契約期間` `作業開始日` `振込手数料負担者` を固定記載、案件固有の追記だけを `備考自由記載` に入れる。
- 相手方の住所、名称、口座、インボイス番号は partner マスタ優先。
- 単一明細では、明細欄に `納期` `支払条件` を重ねて出さない。
